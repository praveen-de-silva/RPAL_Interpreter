1. Introduction

This report describes the design and implementation of a complete interpreter for the RPAL (Right-reference Pedagogic Algorithmic Language) programming language. The interpreter accepts an RPAL source file as input and evaluates it, producing output that matches the reference rpal.exe implementation.

The interpreter is implemented entirely in C++17 without using any lexer/parser generator tools (no lex, yacc, or equivalent). It is built as a six-stage pipeline: Lexer → Screener → Parser → Standardizer → Flattener → CSE Machine.

Usage

./rpal20 <filename>          # evaluate program
./rpal20 -ast <filename>     # print Abstract Syntax Tree and exit
./rpal20 -st  <filename>     # print Standardized Tree and exit

Build

make          # builds the rpal20 executable
make clean    # removes the executable

3. Module Descriptions and Function Prototypes

3.1 Token (lexer/Token.h, Token.cpp)

Defines the TokenType enum and the Token data structure used throughout the pipeline.

enum class TokenType { IDENTIFIER, INTEGER, STRING, OPERATOR, PUNCTUATION, KEYWORD, SPACES, COMMENT, END_OF_FILE }; struct Token { TokenType type; std::string value; Token(TokenType t, std::string v); std::string typeName() const; // returns string name of type };

3.2 Lexer (lexer/Lexer.h, Lexer.cpp)

Stage 1. Reads the entire source file and converts it into a raw token stream (including SPACES and COMMENT tokens that are later removed by the Screener).

class Lexer { public: explicit Lexer(const std::string& filename); // reads file into memory std::vector<Token> tokenize(); // produces token stream private: // Character access char currentChar() const; char peek() const; void advance(); // Character classification bool isLetter (char c) const; bool isDigit (char c) const; bool isOperatorSymbol(char c) const; // Token readers Token readIdentifierOrKeyword(); Token readInteger(); Token readString(); // stores surrounding single quotes in value Token readOperator(); Token readSpaces(); Token readComment(); };

The lexer recognises 21 RPAL keywords: let, in, fn, where, aug, or, not, true, false, nil, dummy, within, and, rec, gr, ge, ls, le, eq, ne.

3.3 Screener (lexer/Screener.h, Screener.cpp)

Stage 2. Filters out SPACES and COMMENT tokens, leaving only meaningful tokens for the parser.

class Screener { public: std::vector<Token> filter(const std::vector<Token>& tokens); };

3.4 ASTNode (parser/ASTNode.h, ASTNode.cpp)

The node type used for both the Abstract Syntax Tree (AST) and the Standardized Tree (ST). Children are stored as a linked list via child/sibling pointers.

class ASTNode { public: std::string type; // node label (e.g. "let", "gamma", "IDENTIFIER") std::string value; // leaf value (e.g. "42", "x", "'hello'") ASTNode* child; // first child ASTNode* sibling; // next sibling ASTNode(std::string t, std::string v = ""); ~ASTNode(); // recursively deletes child subtree void print(int depth = 0) const; // pre-order with dot indentation };

3.5 Parser (parser/Parser.h, Parser.cpp)

Stage 3. Implements a hand-written recursive descent parser for the full RPAL grammar. Uses a tree-building stack; each grammar rule calls buildTree(type, n) to pop n nodes and attach them as children.

class Parser { public: Parser(const std::vector<Token>& tokenStream); ASTNode* parse(); // entry point; returns AST root private: // Token consumption helpers const Token& nextToken() const; void read(const std::string& expectedValue = ""); void buildTree(const std::string& type, int numChildren); // 21 recursive descent functions (one per grammar non-terminal) void parseE(); // Expression void parseEw(); // Expression with where void parseT(); // Tuple void parseTa(); // Tuple augmentation void parseTc(); // Tuple conditional void parseB(); // Boolean void parseBt(); // Boolean term void parseBs(); // Boolean simple void parseBp(); // Boolean primary void parseA(); // Arithmetic void parseAt(); // Arithmetic term void parseAf(); // Arithmetic factor void parseAp(); // Arithmetic primary (handles @ infix) void parseR(); // Rator / Rand application void parseRn(); // Primary (literal, identifier, parenthesised) void parseD(); // Definition void parseDa(); // Definition and void parseDr(); // Definition rec void parseDb(); // Definition basic void parseVb(); // Variable binding void parseVl(); // Variable list };

3.6 Standardizer (standardizer/Standardizer.h, Standardizer.cpp)

Stage 4. Transforms the AST into a Standardized Tree (ST) by applying nine bottom-up rewrite rules. The root node is modified in-place.

class Standardizer { public: ASTNode* standardize(ASTNode* node); // main recursive entry point private: // Helper utilities ASTNode* makeNode(const std::string& type, const std::string& value = ""); ASTNode* copyTree(ASTNode* node); // deep copy (needed for rec rule) ASTNode* getChild(ASTNode* node, int n); // 0-indexed child access int countChildren(ASTNode* node); // Transformation rules void standardizeLet(ASTNode* node); // Rule 1: let x=E in P void standardizeWhere(ASTNode* node); // Rule 2: P where x=E void standardizeFcnForm(ASTNode* node); // Rule 3: f x1 x2 = E void standardizeLambda(ASTNode* node); // Rule 4: multi-param lambda void standardizeAnd(ASTNode* node); // Rule 5: simultaneous defs void standardizeRec(ASTNode* node); // Rule 6: rec (Y* combinator) void standardizeWithin(ASTNode* node); // Rule 7: within void standardizeAt(ASTNode* node); // Rule 8: E @n R (infix) void standardizeEmptyParam(ASTNode* node); // Rule 9: f() = E → dummy };

The nine transformation rules in summary:

Rule 1 (let): let x=E in P → gamma (lambda x P) E
Rule 2 (where): P where x=E → gamma (lambda x P) E
Rule 3 (fcn_form): f x1 x2 = E → = f (lambda x1 (lambda x2 E))
Rule 4 (lambda): lambda x1 x2 E → lambda x1 (lambda x2 E)
Rule 5 (and): and (=x1 E1)(=x2 E2) → = (tau x1 x2) (tau E1 E2)
Rule 6 (rec): rec (= f E) → = f (Y* (lambda f E))
Rule 7 (within): within (=x1 E1)(=x2 E2) → = x2 (gamma (lambda x1 E2) E1)
Rule 8 (@): E @n R → gamma (gamma n E) R
Rule 9 (empty): f () = E → = f (lambda dummy E)

3.7 Flattener (flattener/Flattener.h, Flattener.cpp)

Stage 5. Converts the ST into a list of delta arrays (control structures) consumed by the CSE Machine. Each lambda body and each conditional branch becomes its own numbered delta array.

// Control element types enum class CSENodeType { IDENTIFIER, INTEGER, STRING, OPERATOR, LAMBDA, GAMMA, TAU, DELTA, BETA, ENV, TUPLE, YSTAR, TRUE_VAL, FALSE_VAL, NIL, DUMMY }; class CSENode { public: CSENodeType type; std::string value; // for literals and operators int deltaIndex; // for LAMBDA and DELTA nodes int envIndex; // for ENV nodes std::vector<std::string> boundVars; // for LAMBDA nodes }; class Flattener { public: Flattener(); void flatten(ASTNode* root); const std::vector<std::vector<CSENode>>& getDeltas() const; void print() const; // debug: prints all delta arrays private: void flattenNode(ASTNode* node, int currentDelta); std::vector<std::vector<CSENode>> deltas; int nextDelta; };

3.8 StackValue (cse_machine/StackValue.h, StackValue.cpp)

Represents any value that can appear on the CSE Machine's value stack.

enum class ValueType { INTEGER, STRING, BOOL, NIL, DUMMY, TUPLE, CLOSURE, ETA, BUILTIN, PARTIAL }; class StackValue { public: ValueType type; int intVal; std::string strVal; bool boolVal; int deltaIndex; int envIndex; std::vector<std::string> boundVars; std::vector<StackValue> tupleElems; std::shared_ptr<StackValue> partialArg; // for curried built-ins // Factory methods static StackValue makeInt(int v); static StackValue makeStr(const std::string& v); static StackValue makeBool(bool v); static StackValue makeNil(); static StackValue makeDummy(); static StackValue makeTuple(std::vector<StackValue> elems); static StackValue makeClosure(int deltaIdx, std::vector<std::string> vars, int envIdx); static StackValue makeEta (int deltaIdx, std::vector<std::string> vars, int envIdx); static StackValue makeBuiltin(const std::string& name); static StackValue makePartial(const std::string& builtinName, StackValue firstArg); std::string toString() const; // produces rpal.exe-compatible output string };

3.9 Environment (cse_machine/Environment.h, Environment.cpp)

Represents a single lexical scope in the environment chain. Environments form a linked list (parent chain) enabling lexical scoping.

class Environment { public: Environment(int idx, int parentIdx); void bind(const std::string& name, const StackValue& val); bool lookupLocal(const std::string& name, StackValue& out) const; int getParentIndex() const; private: int index; int parentIndex; std::unordered_map<std::string, StackValue> bindings; };

3.10 CSE Machine (cse_machine/CSEMachine.h, CSEMachine.cpp)

Stage 6. Evaluates the delta arrays using the Control–Stack–Environment model. Implements all 13 CSE rules and 12 built-in functions.

4. CSE Machine – Evaluation Rules

The CSE Machine operates on three data structures:

  • Control (C) – instruction stack holding CSENodes from delta arrays
  • Stack (S) – value stack holding StackValues
  • Environment (E) – current active environment index
RuleTriggerActionAuthor
1IDENTIFIER on controlLookup in env chain; push value to stack230094U
2LAMBDA on controlCreate closure (delta, vars, env); push to stack230094U
3GAMMA + single-param closureNew env, bind one param, push delta body230094U
4GAMMA + multi-param closureNew env, bind params from tuple230094U
5ENV marker on controlRestore environment; preserve return value230094U
6Binary OPERATORPop two values; apply op (+, -, *, /, **, gr, ge, ls, le, eq, ne, or, &, aug); push result230123K
7Unary OPERATOR (neg/not)Pop one value; negate; push result230123K

9TAU(n) on controlPop n values; create tuple; push to stack230123K
10GAMMA + tuple on stackPop integer index; push tuple element (1-based)230123K
